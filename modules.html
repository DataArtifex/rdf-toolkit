Add this resource to an RDF graph.

A nifty method that turn this resource into a stack of RDF triples and adds them to the specified RDF graph.

It makes the following assumptions: - The resource is a dataclass - The dataclass properties are the class attribute name - Attributes starting with an underscore are ignored - The attribute namespace is the same as the resource, unless overridden in the attribute metadata

add_resources(resources, attribute_name: str = None, exact_match: bool = True)[source]

Generic helper to add resources to attributes of this resource.

This is a magic helper to add any resources to an attribute of this resource.

It is recommended to provide the attribute name for faster processing. When the attribute name is not specified, each attribute of the class will be examined for a match based on the resource type. - By default, wee look for an exact resource type match. - This can be adjusted using the exact_match argument, in which case subtypes will also be considered (e.g. a Category for a Concept). - If more than one match is found, an exception will be raised.

If the target attribute is a list, it will be instantiated and the resources will be appended. We do not at this time prevent duplicate list entries (@TODO implement after we have a resource comparison/equality implemented)

If the target attribute is not a list, it will simply be set (replacing any existing value).
